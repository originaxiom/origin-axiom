# Research Atlas

Status: public navigation layer. This directory adds structure around the
governed repository; it does not add claims.

## Purpose

The project now has several kinds of material: proven algebra, conditional
theorems, frontier probes, dead routes, legacy context, source mining, and
long-running campaigns. The Research Atlas makes that work navigable without
rewriting it into false coherence.

The rule is:

```text
raw work stays raw
tested code stays tested
frontier probes stay frontier
the atlas explains how the pieces connect
```

## Source Of Truth

The authoritative ledgers remain:

- `CLAIMS.md` for proven, conditional, open, and dead claims.
- `GOVERNANCE.md` for promotion gates and anti-overclaim rules.
- `PROGRESS_LOG.md` for chronological execution history.
- `frontier/` and `paths/` for bounded probes and verdicts.
- `docs/ARCHIVE.md` for killed or superseded routes.

Atlas pages should never outrun those files. If an atlas page and a governed
ledger disagree, the governed ledger wins.

## Entry Points

- `RESEARCH_TREE.md` gives the current map from the original question to the
  surviving exact spine and the open physics bridge.
- `REVIEWER_GUIDE.md` gives routes for different readers: new reader,
  mathematician, physicist, and reproducibility checker.
- `FAILURE_ATLAS.md` classifies repeated obstruction types so failed routes are
  not repeated.
- `SUCCESS_ATLAS.md` records what survived under governance.
- `GLOSSARY.md` defines recurring terms in project language.
- `SIMULATOR_ECOSYSTEM_MAP.md` defines when an interactive tool is justified.

## Navigation Pattern

Every mature research node should eventually support this path:

```text
Map -> Node -> Evidence -> Code -> Verdict -> Next obstruction
```

That means each important branch should answer:

```text
what question was asked
why it mattered
what object was tested
what proof or code was used
what happened
what verdict was assigned
what mistake this prevents repeating
what remains open
```
