## Base types (the three-layer separation made mechanical)

Every quantity Q constructed from the object O = (M, ρ, A₁) and/or a
stage S_κ receives exactly one base type:

- **T_spec** — spectrum-class: Q is a function of an operator's
  eigenvalue MULTISET (or a predicate on it: closure, gaps, symmetry),
  with no contraction against any external vector or system.
  Observer-independent. (B611 layer 1; chirality lives here, B613.)
- **T_field** — arithmetic-class: Q is a trace, determinant,
  discriminant, conductor-divisibility, field-membership predicate, or
  Galois-orbit datum of an operator or holonomy element. The FIELD and
  orbit structure are object-forced; the position within a Galois
  orbit is stage-chosen (K020). (B611 layer 2.)
- **T_coup(X)** — coupling-class: Q contracts object/stage data
  against an EXTERNAL SYSTEM X — a listener vector u, a second copy or
  partner object, or a chosen stage section. Parameterized by X. The
  campaign thesis ("values live in the coupling") is the license for
  the generality of X; the freeze-set instance is X = a listener
  (B584). Sub-type **T_coup^inv**: provably X-independent within a
  declared X-family (an invariance theorem is required).

## Modifiers (every typed quantity carries all three flags)

1. **Word-class W(w) vs full-image F**: Q depends on a specific
   monodromy word w (W-class) or on the whole representation image
   (F-class).
2. **Core flag**: Q is core-coupled if it factors through the block's
   non-abelian core; character-dressed content is UNTYPEABLE on
   W-class quantities (blind there) and typeable on F-class quantities
   through the core tensor. (B646-Q2, as scoped at the campaign seal.)
3. **Dimension flag**: dimensionless or not. The object carries no
   scale (the banked walls): only dimensionless quantities have
   T-images.

## The typing rules (mechanical; the checker encodes exactly these)

- **R0 (domain; ADDED in the one disclosed revision cycle, 2026-07-16,
  after the held-out failure on B632's v₀)**: the system types
  QUANTITIES — scalar- or predicate-valued constructions.
  Representation-space ELEMENTS (sections, vectors, cohomology
  classes) are CARRIERS, not quantities; a carrier is typed only
  through the quantities built from it (its dimension counts → R1,
  its arithmetic → R2, its contractions → R3). v₀ itself is a
  carrier; "dim h⁰(M; 27) = 1" is its T_spec quantity.
- **R1**: eigenvalue-multiset function, no contraction → T_spec.
- **R2**: tr/det/disc/conductor/field-membership of a single operator
  or holonomy element → T_field.
- **R3**: any contraction against an external system X → T_coup(X);
  with a banked X-independence theorem → T_coup^inv.
- **R4 (comparability)**: Q₁ ~ Q₂ iff same base type AND same all
  three flags; T_coup quantities are cross-system comparable only in
  their T_coup^inv form.
- **R5 (the physics rule)**: an experiment-side observable may be
  compared only to a same-type, dimensionless, core-coupled T-image;
  every comparison names the type judgment in its seal.
- **R6 (one-type)**: a quantity that matches two rules is ill-typed →
  the construction must be decomposed until each piece types uniquely
  (decomposition recorded).
