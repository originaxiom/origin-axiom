## The method note (MB7 — the reducible-locus escape artifact)

A naive search that counts **any** converged fixed point with trace far from `K_m` as an "escape" **over-counts**: the
**reducible** locus (algebra dim < 9) carries degenerate points whose commutator trace `κ` is *not* constrained to
`K_m`, faking escapes. B129's m=1 search did not expose this — its reducible points happen to be **rational** (hence in
ℚ(√−3)); at m=2 the reducible points have messy non-ℚ(i) `κ`, faking ~15–20 "escapes" that are **all reducible**
(algdim 7, `trA=−1`, `|eig|=1`, residual ~1e-15 — well-converged, not under-resolved). 

> **Guard (MB7):** count an escape only among **genuine irreducible** fixed points (algebra generated by A,B = `M₃`,
> dim 9). With that filter the m=2 "escapes" vanish → sealing holds. This is the sibling of K-G / K-H / MB2: a
> surprising "escape" that is a degenerate artifact — run the **irreducibility classification** before counting. (Here
> it nearly read as "S031 is false for m=2"; the filter showed it false-alarm.)
